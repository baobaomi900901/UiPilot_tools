# Launcher Application Icons Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 在启动器应用搜索结果中安全显示 Start Menu `.lnk` 与 AppsFolder 打包应用的真实 Windows 图标，并在任何提取、编码或解码失败时保留应用和显示默认瓦片。

**Architecture:** 沿用现有后台应用扫描与内存 `Application` 快照，在扫描线程用 `IShellItemImageFactory` 获取 32 x 32 HBITMAP，经 WIC 和 Windows Base64 转成受限 PNG data URL，再复用既有 `Application.icon -> ResultItem.icon -> protocol icon?: string` 链路。前端 core 只保留严格合法且不超过 64 KiB 的 PNG data URL，视图在固定 28 x 28 容器中互斥显示真实图标或 CSS 默认瓦片，不增加命令、缓存、状态库或线程。

**Tech Stack:** Rust 1.96.0、`windows` 0.61.3、Tauri 2.11.3、React 19.2.7、TypeScript 7、Vitest 4、Windows Shell/GDI/WIC/Crypt32。

## Global Constraints

- 唯一产品边界是 `docs/superpowers/specs/2026-07-21-launcher-application-icons-design.md`；不修改该规格的状态或扩展其范围。
- 实现起点是本计划分支 merge commit `f81f141364a89c5913c6d151a40f9102c8d53958`，其中本地权威 `main` 为 `a8f19cb76577ce6b4348e83bdace32ea4c88b601`。
- 继续锁定 `windows = 0.61.3`；只增加 `Win32_Graphics_Gdi`、`Win32_Graphics_Imaging`、`Win32_System_Com_StructuredStorage` features，不增加 crate 或 npm 依赖。
- 只允许 `data:image/png;base64,<payload>`；总 ASCII 长度不超过 `65,536`，payload 非空、长度为 4 的倍数、字符只来自标准 Base64 字母表，`=` 只可在尾部出现一个或两个。

- 复用共享 `ResultRegistry`；不得改变 `QueryDomain::File`、`OpenIndexedPath`、跨域失效或文件搜索 DTO。
- 不渲染 Settings UI 图标；不增加磁盘缓存、懒加载、独立图标命令、线程池、trait、factory、远程资源、SVG 或路径 fallback。

```rust
const ICON_EDGE: i32 = 32;
const DATA_URL_PREFIX: &str = "data:image/png;base64,";
const MAX_DATA_URL_BYTES: usize = 65_536;
const MAX_PNG_BYTES: usize = 49_134;

pub(super) fn from_shortcut(path: &Path) -> Option<String> {
    let path: Vec<u16> = path.as_os_str().encode_wide().chain([0]).collect();
    let item: IShellItem = unsafe {
        SHCreateItemFromParsingName(PCWSTR(path.as_ptr()), None::<&IBindCtx>)
    }
    .ok()?;
    from_shell_item(&item)
}

pub(super) fn from_shell_item(item: &IShellItem) -> Option<String> {
    let factory: IShellItemImageFactory = item.cast().ok()?;
    let bitmap = unsafe {
        factory.GetImage(
            SIZE { cx: ICON_EDGE, cy: ICON_EDGE },
            SIIGBF_ICONONLY,
        )
    }
    .ok()?;
    with_owned_bitmap(bitmap, bitmap_png_data_url)
}

fn with_owned_bitmap<T>(
    bitmap: HBITMAP,
    encode: impl FnOnce(HBITMAP) -> Option<T>,
) -> Option<T> {
    let bitmap = unsafe { Owned::new(bitmap) };
    encode(*bitmap)
}
```

WIC 函数保持一个线性 `Option` 流程：创建 `IWICImagingFactory`，以 `WICBitmapUseAlpha` 包装 HBITMAP，创建大小固定为 `MAX_PNG_BYTES` 的 `IWICStream`，用 `GUID_ContainerFormatPng`、`WICBitmapEncoderNoCache`、32 x 32 frame 与 `GUID_WICPixelFormat32bppBGRA` 写入；frame/encoder commit 后通过 `Seek(0, STREAM_SEEK_CUR, ...)` 取得实际长度。任何 `.ok()?`、缺少 frame、像素格式被替换、零长度或越界均返回 `None`。

```rust
fn bitmap_png_data_url(bitmap: HBITMAP) -> Option<String> {
    let factory: IWICImagingFactory = unsafe {
        CoCreateInstance(&CLSID_WICImagingFactory, None, CLSCTX_INPROC_SERVER)
    }
    .ok()?;
    let source = unsafe {
        factory.CreateBitmapFromHBITMAP(bitmap, HPALETTE::default(), WICBitmapUseAlpha)
    }
    .ok()?;
    let bytes = vec![0_u8; MAX_PNG_BYTES];
    let written = {
        let stream = unsafe { factory.CreateStream() }.ok()?;
        unsafe { stream.InitializeFromMemory(&bytes) }.ok()?;
        let encoder = unsafe {
            factory.CreateEncoder(&GUID_ContainerFormatPng, std::ptr::null())
        }
        .ok()?;
        unsafe { encoder.Initialize(&*stream, WICBitmapEncoderNoCache) }.ok()?;
        let mut frame = None;
        unsafe { encoder.CreateNewFrame(&mut frame, std::ptr::null_mut()) }.ok()?;
        let frame = frame?;
        unsafe { frame.Initialize(None::<&IPropertyBag2>) }.ok()?;
        unsafe { frame.SetSize(ICON_EDGE as u32, ICON_EDGE as u32) }.ok()?;
        let mut format = GUID_WICPixelFormat32bppBGRA;
        unsafe { frame.SetPixelFormat(&mut format) }.ok()?;
        if format != GUID_WICPixelFormat32bppBGRA {
            return None;
        }
        unsafe { frame.WriteSource(&source, std::ptr::null()) }.ok()?;
        unsafe { frame.Commit() }.ok()?;
        unsafe { encoder.Commit() }.ok()?;
        let mut written = 0_u64;
        unsafe { stream.Seek(0, STREAM_SEEK_CUR, Some(&mut written)) }.ok()?;
        usize::try_from(written)
            .ok()
            .filter(|length| (1..=MAX_PNG_BYTES).contains(length))?
    };
    png_data_url(&bytes[..written])
}
```

`png_data_url` 使用现有 Crypt32 两次调用，flags 固定为 `CRYPT_STRING_BASE64 | CRYPT_STRING_NOCRLF`，去掉 API 返回缓冲的终止 NUL，严格 UTF-16 转换并重验最终长度；不写自制 Base64：

```rust
fn png_data_url(png: &[u8]) -> Option<String> {
    if png.is_empty() || png.len() > MAX_PNG_BYTES {
        return None;
    }
    let flags = CRYPT_STRING(CRYPT_STRING_BASE64.0 | CRYPT_STRING_NOCRLF);
    let mut length = 0_u32;
    if !unsafe { CryptBinaryToStringW(png, flags, None, &mut length) }.as_bool() {
        return None;
    }
    let mut encoded = vec![0_u16; length as usize];
    if !unsafe {
        CryptBinaryToStringW(
            png,
            flags,
            Some(PWSTR(encoded.as_mut_ptr())),
            &mut length,
        )
    }
    .as_bool()
    {
        return None;
    }
    let end = encoded.iter().position(|unit| *unit == 0).unwrap_or(encoded.len());
    let payload = String::from_utf16(&encoded[..end]).ok()?;
    let result = format!("{DATA_URL_PREFIX}{payload}");
    (result.len() <= MAX_DATA_URL_BYTES).then_some(result)
}
```

const ICON_PREFIX = 'data:image/png;base64,'
const MAX_ICON_LENGTH = 65_536
const BASE64 = /^(?:[A-Za-z0-9+/]{4})*(?:[A-Za-z0-9+/]{2}==|[A-Za-z0-9+/]{3}=)?$/

function safeApplicationIcon(value: unknown): string | undefined {

在 `apps/cache.rs` 测试模块使用同步 channel；测试模块可直接 `try_read` 私有 `applications` 锁，避免 timeout 或调度猜测：

```rust
#[test]
fn blocked_discovery_does_not_hold_the_snapshot_lock() {
    let cache = Arc::new(AppCache::from_apps(vec![application("Existing")]));
    let (entered_tx, entered_rx) = mpsc::sync_channel(0);
    let (release_tx, release_rx) = mpsc::sync_channel(0);
    let worker_cache = Arc::clone(&cache);
    let worker = thread::spawn(move || {
        worker_cache
            .refresh_with(|| {
                entered_tx.send(()).unwrap();
                release_rx.recv().unwrap();
                Ok(snapshot(vec![application("New")]))
            })
            .unwrap();
    });
    entered_rx.recv().unwrap();
    assert!(cache.applications.try_read().is_ok());
    assert_eq!(titles(&cache.snapshot()), ["Existing"]);
    release_tx.send(()).unwrap();
    worker.join().unwrap();
    assert_eq!(titles(&cache.snapshot()), ["New"]);
}
```
