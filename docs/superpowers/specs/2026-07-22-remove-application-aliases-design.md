# 移除应用别名功能设计

## 背景

设置页当前加载全部应用，并为每个应用提供别名增删和保存能力。应用别名不属于当前产品需求，需要从界面、协议、持久化和搜索行为中完整删除，而不是仅隐藏设置页控件。

## 目标

- 设置页不再显示应用名称、别名输入框或别名操作按钮。
- 设置加载与保存协议不再包含应用列表或 `aliases` 字段。
- Rust 设置模型不再保存、校验或装饰应用别名。
- 应用搜索不再读取或匹配别名。
- 保留快捷键、Research ID、开机启动、使用次数和其他现有设置行为。
- 不修改插件系统及 `/math` 功能。

## 非目标

- 不新增替代别名功能。
- 不新增设置迁移器或设置 schema 版本。
- 不清除或重置用户的其他设置。

## 设计

### 前端

`SettingsSnapshot` 只保留设置页实际使用的字段。删除应用别名视图类型、私有应用状态、应用 ID 映射、别名输入控件所有权，以及新增和删除别名的方法。

设置页删除应用列表整段渲染和对应 CSS。保存请求只提交快捷键、开机启动和可选 Research ID。

### IPC

`SettingsView` 不再返回应用列表，`UserSettingsUpdate` 不再接收 `aliases`。加载设置不再遍历应用缓存，保存设置不再为了别名校验依赖应用缓存。

### Rust 设置与搜索

从 `Settings`、`SettingsUpdate` 和 `Application` 删除别名字段。删除别名校验、应用装饰、更新合并及排序匹配逻辑。应用搜索继续使用名称、使用次数和现有稳定排序规则。

Serde 默认忽略旧 `settings.json` 中的未知 `aliases` 字段，因此无需保留别名兼容代码。后续任何设置写入都会按新结构序列化，旧字段自然消失。

## 错误处理

删除别名后，设置加载不应因旧文件中存在 `aliases` 而失败。快捷键、Research ID、开机启动和使用次数仍沿用现有校验及原子持久化错误处理。

## 测试

- 前端视图测试确认设置页不显示应用名称、别名文本或别名按钮。
- 前端保存测试确认请求不含 `aliases`。
- Rust 序列化测试确认设置视图和更新协议不含别名字段。
- Rust 设置测试确认旧 JSON 中的 `aliases` 可读取，重新保存后字段消失，其他设置保留。
- 排序测试确认搜索仅依赖应用名称等保留字段。
- 运行完整前端测试、Rust 测试、构建和 Clippy。

## 验收标准

1. 打开设置页时不出现任何应用列表或别名编辑结构。
2. 保存设置后界面仍不出现应用列表。
3. 项目代码的数据结构和业务逻辑中不再存在应用别名能力。
4. 旧设置文件不会阻止程序启动或设置加载。
5. 应用搜索、插件搜索和 `/math` 行为保持正常。
