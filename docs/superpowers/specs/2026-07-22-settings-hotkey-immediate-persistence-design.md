## 背景与根因

设置页的录制器在一次有效录制完成后只调用前端 `setHotkeyCanonical`。该调用更新设置模型中的快捷键草稿，但不调用 Rust 命令。现有 `saveSettings` 是设置页“保存”按钮唯一的持久化入口，并发送完整的 `hotkey`、`autostart`、`researchId` 与 aliases 快照。因此，录制 `DoubleCtrl` 后没有再点保存时，磁盘保留旧值，运行时也继续保留旧绑定；这两个现象来自同一状态边界。

诊断已确认：

- 前端测试锁定“录制 canonical setter 不保存”，而全量保存会发送 `DoubleCtrl`。
- `SettingsStore` 的原子持久化、`LifecycleCoordinator` 的热键事务/补偿、启动 reconcile、double-tap detector 和 hook 单元测试均通过。
- 运行时只通过 Tauri `app_data_dir()` 加载 `com.uipilot.launcher` 的 `settings.json`；旧 `%LOCALAPPDATA%\UiPilot` 数据不在当前加载链中。

## 目标

1. 录制成功后立即且仅持久化、激活快捷键。
2. `DoubleCtrl`、`DoubleAlt` 和普通 chord 都在当前运行中立即生效，并在重启后从持久化设置一致恢复。
3. 录制的即时动作绝不持久化同页 aliases、Research ID、autostart 或其他未保存草稿。
4. 失败不假装成功：界面回到最后已持久化的快捷键，其他草稿保留，并显示固定本地错误。
5. 后续全量“保存”使用已更新的 durable hotkey，不能把旧值写回磁盘。

## 非目标

- 不改变全量“保存”负责的其他设置语义。
- 不新增通用 settings patch 框架、设置缓存、依赖、日志、迁移或旧 `%LOCALAPPDATA%\UiPilot` 数据读取。
- 不变更 result、启动器、文件搜索、协议中无关字段或专用 hook 线程。
- 不把 AutoGenerated AppsFolder、快捷键冲突的产品策略或任意设置字段合并进本次范围。

## UX 与状态机

设置模型区分“最后 durable hotkey”和所有仍可编辑的表单草稿。录制器给出 canonical 值后，不经过全量保存，而是启动一个 `hotkey` 专用 settings operation。

| 状态 | 触发 | 界面与操作 |
| --- | --- | --- |
| 可编辑 | 录制前 | 快捷键和其他字段都是草稿；全量保存可用。 |
| 快捷键提交中 | 录制 commit | 快捷键录制、重复录制和全量保存禁用；其他草稿不序列化。 |
| 成功 | 专用命令返回 canonical 值 | 仅 hotkey 的 durable 值和显示更新；其他草稿原样保留；全量保存重新可用。 |
| 失败且可确定 | 命令返回失败且事务已回滚 | hotkey 显示恢复为提交前 durable 值；其他草稿不变；显示现有固定 settings 错误。 |
| 状态不确定 | 事务/回滚无法确认 | hotkey 恢复到本地已知 durable 值，其他草稿保留在内存；设置页 fail closed，要求重新加载或重启后才允许继续写入。 |

现有单一 `settingsOperation` token 仍是所有权边界。专用 operation 进行期间，新的录制 commit 与全量保存都不启动；视图切换、重载或较新 operation 使旧 operation 过期后，其成功/失败响应不得改写新 settings view。

## 前端与 IPC 数据流

1. `HotkeyRecorderInput` 从 `reduceHotkeyRecorder` 得到 canonical commit。
2. core 先验证 settings 可编辑且不存在 settings operation，保存当前 durable hotkey 作为回滚值，再发起专用客户端调用，例如 `saveHotkey({ hotkey })`。
3. IPC payload 只含 hotkey 字符串；不携带 aliases、Research ID、autostart、应用 ID 或其他草稿。
4. 成功响应只返回 canonical durable hotkey。core 仅更新 hotkey 控件的 value/draft，并释放 operation；`settingsUpdate()` 因而在后续全量保存时自然使用新值。
5. 失败响应不得暴露 Rust、路径或 hook 原始错误。core 使用既有固定 settings 错误映射，恢复保存前 hotkey，保留其他草稿；不确定状态额外标记 `needsReload`/只读。

专用操作不能直接调用当前全量 `saveSettings`。后者基于完整 settings snapshot 构造 payload，录制时调用会把尚未确认的 aliases、Research ID 或 autostart 一并写盘，违反本规格的隔离要求。

## Rust 最小边界

新增一个最小专用 Tauri 命令和对应前端 client 方法，只服务第二个真实调用方：即时 hotkey 保存。

- 命令先以现有 `HotkeyKind::parse` 验证输入并取得 canonical 值。
- 命令使用现有 `CriticalReservation`、worker 调度与 `LifecycleCoordinator`，避免与全量保存、退出或另一 settings operation 交错。
- `LifecycleCoordinator` 新增窄的 hotkey-only 事务入口，复用现有 `apply_hotkey_settings_transaction`、main-thread hook 安装/卸载、global shortcut 注册、补偿和持久化最后的不变量；不复制完整生产热键事务。
- `SettingsStore` 新增窄的 hotkey-only 原子更新：在锁内从当前持久化 `Settings` 克隆，仅替换 canonical hotkey，经既有原子文件路径写入成功后才发布内存状态。它不接受泛型字段集合，也不验证或重写其他用户字段。

全量 `save_settings` 保持现有 payload 和责任。它从前端当前 hotkey 控件构造 update，因此即时成功后会自然提交最新 durable hotkey。

## 事务顺序与补偿不变量

专用事务从 `SettingsStore::snapshot()` 读取旧 durable settings 与旧 `HotkeyKind`，并构造只改变 hotkey 的持久化闭包。

1. 按现有热键事务注册新 chord 或安装目标 double-tap hook；必要时卸载/注销旧运行时绑定。
2. 保持现有 autostart 实际值，不将前端 autostart 草稿带入事务。
3. 最后调用 hotkey-only `SettingsStore` 原子持久化。
4. 只有全部成功后，运行时、磁盘和前端 durable hotkey 才一起前进。

任一失败时，持久化尚未发生或原子写入失败，运行时按现有快照补偿恢复旧 shortcut/hook/autostart。每个成功的平台操作才更新 runtime 镜像；补偿中的独立清理继续执行，避免残留新 chord 或 hook。若补偿或句柄卸载无法确认，运行时镜像必须反映观察到的实际状态，命令返回失败，前端 fail closed 并要求重载或重启。磁盘不得在运行时后续仍可失败的阶段提前写入。

## 并发与陈旧响应

- 录制 commit 通过现有 `settingsOperation` 取得唯一 token，pending 时拒绝重复 commit、全量保存和其他写型 settings action。
- IPC worker 保留现有 critical reservation，避免退出与持久化/热键切换交错。
- 前端仅在 token、view epoch 和 settings view 都仍属于该 operation 时处理响应；陈旧成功或失败只释放其所有权，不能覆盖新 settings view。
- 成功后仅更新 hotkey 控件，不调用全量 reload，避免覆盖用户仍在编辑的 aliases、Research ID 或 autostart 草稿。

## 错误处理

输入非法、reservation 失败、dispatcher 拒绝、chord 注册失败、hook 安装/卸载失败、autostart 状态检查失败或原子持久化失败均返回既有固定 settings 失败语义。前端不显示 backend 文本。

确定回滚失败时，前端显示提交前 durable hotkey 并保留其他草稿。若 coordinator 无法确认运行时已恢复，前端还进入只读待重载状态；重启会按磁盘 durable hotkey reconcile，避免把不确定状态继续暴露为可编辑成功。

## 人工验收

1. 打开设置，录制 DoubleCtrl，不点击全量“保存”；隐藏主界面后双击 Ctrl，Launcher 立即显示。
2. 重启应用后再次双击 Ctrl，确认仍能显示 Launcher。
3. 在同页修改 alias、Research ID 或 autostart 但不点击全量保存，录制另一快捷键；重开设置确认只有快捷键已 durable，其他字段未被即时动作写盘。
4. 录制 Alt+Space，确认当前运行立即切换并在重启后保留。
5. 制造可控注册或 hook 失败，确认界面不显示成功，hotkey 回到旧 durable 值，其他草稿仍保留；若提示需要重载/重启，操作后检查磁盘与运行时一致。

## 风险与拒绝方案

- **直接调用全量 `saveSettings`：拒绝。** 会误保存不相关草稿，数据丢失/误提交风险最高。
- **通用 settings patch 命令：拒绝。** 只有 hotkey 是第二个真实调用方，通用字段补丁会扩大验证、并发和测试面。
- **只改前端，不做持久化：拒绝。** 重启后仍恢复旧绑定，不能满足即时且 durable 的需求。
- **延后到用户点击保存：拒绝。** 这正是当前根因。
- **迁移旧 `%LOCALAPPDATA%\UiPilot` 数据：拒绝。** 已被诊断排除，且会扩大隐私与兼容范围。

主要实现风险是运行时绑定、磁盘和前端三个状态面分叉。通过持久化最后、现有补偿、专用 payload、单 operation token 和不确定状态 fail closed 控制该风险。
